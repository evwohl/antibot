The Importance of Stopping Bots

By Evan Wohl

A

Bots can cause unwanted stress on a website's server. This stress can eventually take down a website in what is known as a Denial of Service attack. Stopping artificial traffic from reaching a website with an anti-bot and captcha system can greatly minimize the chance of the attack being successful.

B

Bots can carry out bruteforce attacks to steal user information. With many data breaches floating around the internet, it is quite common for people to attempt to bruteforce accounts on the internet using old data. Therefore, an antibot system can keep a website safe from attackers hungry for user information.

C

Bots can take away from the experience of real users. Within the past few years, many people have started to create bots and scripts to automatically complete tasks on a website that are profitable, such as buying a pair of shoes that are limited edition and then selling them for a profit on the secondary market.

Methods

In order to stop bots from overloading websites and stealing information, many companies are starting to incorporate cookie-based anti bot systems. These systems involve employing a variety of methods to detect and stop bot traffic. In my model for an antibot system, I used the following methods to stop bots.

Captcha
Simplistic yet efficient. An image is created with text, and the user responds with the text in the image.
Click detection
The JavaScript code detects clicks and compares them with mouse events to ensure the clicks are coming from a human.
Mouse Movement
The code also tracks mouse movement and compares it with earlier movement. In the future, I hope to be able to create a neural network and use AI to detect fake traffic.

About the project

I have spent the past few months learning JavaScript, HTML, and CSS to create this website, along-side a bot detection and captcha algorithm that aids in stopping artificial traffic from entering/automatically completing tasks on a website. The obfuscated JavaScript code is on Github along with the rest of the projects code.

The project works by giving each user a trust score esentially. A user enters a website and they are immediately untrusted and prompted to solve a captcha. Doing so will bring their score to 50%. From there on out, the way a user interacts with a website determines their score. If their mouse movement pixels dont match up with the click pixels, their score goes down. If their mouse jumps too fast, the score goes down. Other actions such as scrolling increase trust score.
